# Model Card — Former-It

## Model details
- **Type:** a from-scratch tiny **Transformer encoder** — token embedding +
  sinusoidal positional encoding → stacked pre-norm blocks (multi-head
  self-attention + feed-forward, each with a residual connection and layer norm)
  → mean-pool → linear classifier — trained with Adam + cross-entropy loss
  (PyTorch). Built without `nn.TransformerEncoder` so the internals are legible.
- **Size:** `d_model=32`, `n_heads=2`, `n_layers=2`, `d_ff=64` by default.
- **Explainability:** every prediction returns the **per-layer, per-head
  self-attention matrices** (`[n_layers][n_heads][T][T]`, each row a softmax
  distribution). The UI renders them as heatmaps so a viewer can see *which
  positions attend to which*.
- **Training:** per-run hyperparameters (learning rate, batch size, epochs)
  supplied by the API request.
- **Owner:** your team (add your names).

## Intended use
- **Primary:** an interactive teaching demo of self-attention, multi-head
  Transformer encoders, and attention visualisation across three clouds.
- **Out of scope:** any real-world NLP task. The model is trained on
  **synthetic** symbol sequences with a known algorithmic rule.

## Data
- Synthetic integer-symbol sequences (alphabet of 5 symbols, default length 8),
  classified as **palindrome** vs **random**. Palindromes are built by mirroring
  a random half; non-palindromes are drawn at random (rejecting any that happen
  to be palindromic), so labels are exact and balanced.
- Generated by `shared/data.generate_sequences` deterministically from a seed;
  only the generation **parameters** (`n_rows`, `seq_len`, `seed`) are stored in
  the Supabase `datasets` table — never the sequences.
- **No real text or personal data** are used or stored.

## Metrics
- Reported on a held-out 20% test split: **accuracy** and **macro-F1**
  (computed manually, no sklearn dependency in the API). The 50% binary baseline
  makes the accuracy floor meaningful.
- Every run's metrics are persisted to the Supabase `runs` table (single source
  of truth — not recomputed in the UI).

## Privacy — what we log (and don't)
- For every served prediction we store the sequence's **sha256 hash + length +
  predicted class + confidence** in `sequence_metadata` — **never the raw
  symbols** (the "store hashes, not raw payloads" invariant).
- The fitted model is stored (base64-pickled) in `run_artifacts`, which is
  **not** anon-readable; only run metrics are exposed to the UI's anon key.

## Limitations
- A compact Transformer on a toy palindrome task; it will not generalize to
  natural language.
- Attention heatmaps are a helpful window into the model, but attention is not a
  guaranteed causal explanation — a head may spread attention or route
  information indirectly through residual connections.
- Sensitive to learning rate and epochs: too few epochs under-fits and the
  attention stays diffuse instead of forming the anti-diagonal.

## Ethical considerations
- A non-technical client must be told the model's limits plainly
  (honesty/stewardship): it is a demonstration on synthetic data, not a validated
  NLP system. The per-head heatmaps build intuition by showing where the model
  looks, but a striking pattern is not proof of correctness — disclose the
  held-out metrics and caveats.
