# Model Card — Attend-It

## Model details
- **Type:** an embedding → single-layer LSTM → **additive (Bahdanau) attention**
  → linear classifier, trained with Adam + cross-entropy loss (PyTorch) for
  multi-class sequence classification.
- **Explainability:** the attention layer scores each LSTM hidden state
  `e_t = vᵀ tanh(W h_t)`, softmaxes the scores into weights `α_t` that sum to 1,
  and forms a context vector `c = Σ_t α_t h_t`. The per-timestep weights `α_t`
  are returned with every prediction so a non-technical viewer can see *which
  token* the network relied on.
- **Training:** per-run hyperparameters (learning rate, batch size, epochs)
  supplied by the API request.
- **Owner:** your team (add your names).

## Intended use
- **Primary:** an interactive teaching demo of a recurrent classifier, additive
  attention as built-in explainability, and the train → evaluate → serve →
  explain loop across three clouds.
- **Out of scope:** any real-world NLP task (content moderation, sentiment on
  real text, etc.). The model is trained on **synthetic** token sequences with a
  known generating rule.

## Data
- Synthetic integer-token sequences (vocab of 24 ids, default length 12), one of
  four classes — `alpha`, `beta`, `gamma`, `delta`. Each sequence contains one
  `TRIGGER` token; the token immediately after it is a **class marker** encoding
  the label. Every other slot is a filler drawn from a disjoint vocabulary range.
- Generated by `shared/data.generate_sequences` deterministically from a seed;
  only the generation **parameters** (`n_rows`, `seq_len`, `seed`) are stored in
  the Supabase `datasets` table — never the token sequences.
- **No real text or personal data** are used or stored.

## Metrics
- Reported on a held-out 20% test split: **accuracy** and **macro-F1**
  (computed manually, no sklearn dependency in the API).
- Every run's metrics are persisted to the Supabase `runs` table (single source
  of truth — not recomputed in the UI).

## Privacy — what we log (and don't)
- For every served prediction we store the sequence's **sha256 hash + length +
  predicted class + confidence** in `sequence_metadata` — **never the raw
  tokens** (the "store hashes, not raw payloads" invariant).
- The fitted model is stored (base64-pickled) in `run_artifacts`, which is
  **not** anon-readable; only run metrics are exposed to the UI's anon key.

## Limitations
- A compact LSTM on a toy trigger-token task; it will not generalize to natural
  language.
- Attention weights are a useful, faithful-*ish* signal for this task — because
  the label genuinely depends on one marked position — but in general attention
  is not a guaranteed causal explanation of a model's decision.
- Sensitive to learning rate and epochs: too few epochs under-fits and the
  attention spreads out instead of localizing on the marker.

## Ethical considerations
- A non-technical client must be told the model's limits plainly
  (honesty/stewardship): it is a demonstration on synthetic data, not a validated
  NLP system. The attention bar builds trust by showing which token mattered, but
  a plausible-looking weighting is not proof of correctness — disclose the
  held-out metrics and caveats.
