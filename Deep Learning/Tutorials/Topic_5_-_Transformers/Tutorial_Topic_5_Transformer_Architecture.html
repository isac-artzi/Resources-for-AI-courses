Topic 5 — Transformer Architecture: Encoder, Decoder, Encoder–Decoder

A bottom-up tour of the transformer block, from layer normalization through multi-head attention to the three archetypes and the decoding strategies that actually generate text.

What you should walk away with. (1) Fluent with the encoder block: LayerNorm → MultiHeadAttention → residual → LayerNorm → MLP → residual. (2) Able to compute multi-head shapes by hand. (3) Comfortable with sinusoidal, learned, and rotary positional encodings. (4) Able to pick the right archetype (encoder, decoder, encoder–decoder) for a task on sight. (5) Familiar with greedy, top-k, and top-p generation; know what temperature does. (6) Aware of the KV cache and why it makes generation tractable.

1 · Layer normalization mathinteractive

LayerNorm normalizes activations across the feature dimension within each example independently:

μ = mean over d_model of x, σ² = variance over d_model of x
out = γ · (x − μ) / √(σ² + ε) + β

Three things that distinguish LayerNorm from BatchNorm:

  • Per example. The mean and variance are computed within each example's feature vector — no dependence on the batch.
  • Works with batch = 1. No batch-dependent statistics; behaves the same at training and inference.
  • Suitable for variable-length sequences. Each position is normalized on its own.

Demo · LayerNorm in action

The bar chart shows input feature values; the orange overlay is the LayerNorm output. Try changing γ and β and watch the curve scale and shift.

2 · The encoder block, end to end interactive

A single transformer encoder block has six layers in a strict order. With pre-norm (the modern default):

x_in → LayerNorm → MultiHeadAttention → + (residual from x_in) → x_mid
x_mid → LayerNorm → MLP → + (residual from x_mid) → x_out

The MLP is per-position and is usually a two-layer feed-forward with an expansion factor of 4:

MLP(x) = Linear(d_model → 4·d_model) → GELU → Linear(4·d_model → d_model)

Diagram · The encoder block

All connections preserve the shape (B, T, d_model). Residuals carry the original input forward; LayerNorm stabilizes the inputs of each sublayer.

3 · Multi-head attention, with shapes mathinteractive

Multi-head attention projects Q, K, V each from (B, T, d_model) into H heads of dimension d_k = d_model / H:

x:          (B, T, d_model)
Q = x @ W_Q (B, T, d_model)        →  reshape (B, T, H, d_k)  →  transpose (B, H, T, d_k)
K = x @ W_K (B, T, d_model)        →  reshape (B, T, H, d_k)  →  transpose (B, H, T, d_k)
V = x @ W_V (B, T, d_model)        →  reshape (B, T, H, d_k)  →  transpose (B, H, T, d_k)

attn = softmax(Q @ Kᵀ / √d_k)      (B, H, T, T)
out  = attn @ V                    (B, H, T, d_k)
out  = transpose + reshape         (B, T, d_model)
out  = out @ W_O                   (B, T, d_model)

4 · Different heads, different patterns interactive


One of the empirical findings of "Attention Is All You Need" and follow-up interpretability work: different heads specialize. Some heads attend to the previous token (a local pattern); others attend to syntactic dependencies (e.g., the verb's subject); others attend uniformly. Multi-head wins not because of "more parameters" but because of parallel specialization.

5 · Sinusoidal positional encoding mathinteractive


Self-attention is permutation-equivariant: it treats the input as a set. To restore order, positional encoding adds a position-dependent vector to each token's embedding:



  PE(pos, 2i) = sin(pos / 100002i/d)

  PE(pos, 2i+1) = cos(pos / 100002i/d)



The frequencies decay geometrically with the dimension index, so low dimensions encode fine-grained position information and high dimensions are nearly constant across the sequence. Two practical properties:


    
  
  • Different positions get different vectors, so the network can recover order.
    • 
  
  • Because sin(a + b) = sin a cos b + cos a sin b, the encoding of relative offsets can be expressed as a linear function of the absolute encodings — useful for the model to learn shift-invariant patterns.
    •

6 · Learned vs sinusoidal vs rotary math



  
SchemeHow position enters the modelProsCons

  
Sinusoidal (fixed)Add a fixed sin/cos vector to the token embeddingNo parameters; some extrapolation to longer sequencesLimited expressiveness; bounded effective context

  
LearnedAdd a trainable (max_len, d_model) table indexed by positionMost expressive; matches the dataCannot extrapolate past max_len

  
Rotary (RoPE)Rotate Q and K in 2-D pairs by angles that depend on position; the dot product then depends only on relative positionClean relative-position semantics; extrapolates further than learnedMore complex implementation; needs to be applied inside attention rather than added to embeddings






7 · Pre-norm vs post-norm math


The original "Attention Is All You Need" uses post-norm: x_out = LayerNorm(x_in + Sublayer(x_in)). Modern transformers (GPT-2 onward) almost all use pre-norm: x_out = x_in + Sublayer(LayerNorm(x_in)).


The difference matters more in deeper networks:


    
  
  • Post-norm: residual is normalized — gradients can compress or amplify across layers; deep stacks usually require careful learning-rate warmup to train.
    • 
  
  • Pre-norm: the residual path remains an "identity highway" that is not normalized — gradients flow cleanly back. Higher learning rates work without warmup; very deep stacks (24, 48, 96 layers) trainable.
    •

9 · Cross-attention math


Inside an encoder–decoder transformer's decoder, two attention sublayers sit side by side:


    
  
  1. Self-attention with a causal mask over the decoder's own positions.
    2. 
  
  2. Cross-attention where Q comes from the decoder hidden state but K and V come from the encoder's output. No causal mask is applied here — the decoder is allowed to look at every input position.
    3. 



This is the architectural primitive behind translation, summarization, and any seq2seq task: each generated token can directly attend to every source-side token.




10 · Three archetypes side by side math



  

    
Encoder-only · BERT

    
Bidirectional self-attention. Pre-trained with masked-language-modeling. Strong for classification and embedding tasks.

  

  

    
Decoder-only · GPT

    
Causal self-attention. Pre-trained with next-token prediction. Strong for open-ended generation and in-context prompting.

  

  

    
Encoder–Decoder · T5 / BART

    
Encoder reads the source bidirectionally; decoder generates causally; cross-attention grounds output in source. Strong for conditional sequence generation.

11 · Parameter count of a transformer mathinteractive


For a transformer with L layers, d_model, MLP expansion factor 4, vocab size V:



  per-layer ≈ 4 · d_model² (attention proj) + 8 · d_model² (MLP) = 12 · d_model²

  total ≈ L · 12 · d_model² + V · d_model



For GPT-2 small: L = 12, d_model = 768, V ≈ 50,000. Plug in: 12 · 12 · 768² + 50000 · 768 ≈ 85 M + 38 M ≈ 125 M.

12 · KV cache for fast decoding math


At inference time, decoder-only transformers generate one token per step. Naive implementation: at each step, recompute K and V for every previous token. That is O(t²) per step and O(T³) for the whole sequence.


The KV cache stores K and V from previous steps so each new step only needs to compute K and V for the one new token, then attend to the cached past:


at step t (new token):
  q = X[t] · W_Q                                # shape (1, d_k)
  k_new = X[t] · W_K  → append to kv_cache[K]   # cache grows by one
  v_new = X[t] · W_V  → append to kv_cache[V]   # cache grows by one
  out = softmax(q @ kv_cache[K]ᵀ / √d_k) · kv_cache[V]    # attend over cached past


Cost per token drops from O(t²) to O(t); total sequence cost from O(T³) to O(T²). Without the KV cache, modern long-context LLMs would be unusably slow.




13 · Generation strategies math


Generation turns a probability distribution over the vocabulary at each step into actual tokens. Four standard strategies:



  
StrategyHowTrade-off

  
GreedyPick the argmax at every stepFastest; deterministic; tends to be repetitive

  
Beam searchKeep top-K parallel hypothesesBetter likelihood; still safe / generic outputs

  
Top-k samplingSample from the top K logits, renormalizedDiverse; quality depends on K

  
Top-p (nucleus)Sample from the smallest set of tokens whose cumulative probability ≥ pAdaptive: small set when distribution is sharp, large when uncertain
