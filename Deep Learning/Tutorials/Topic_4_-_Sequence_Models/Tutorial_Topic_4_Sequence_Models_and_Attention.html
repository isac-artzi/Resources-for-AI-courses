Topic 4 — Sequence Models & the Attention Mechanism

From a vanilla RNN unrolled in time, through LSTM gates and GRUs, to scaled-dot-product attention and its built-in interpretability surface. By the end you should be able to compute attention by hand and reason about LSTM-vs-attention trade-offs without consulting a table.

What you should walk away with. (1) How a vanilla RNN passes information through time and why its gradients vanish or explode. (2) The role of each LSTM gate and the difference between an LSTM and a GRU. (3) Tokenization → embedding → sequence-model input, end to end. (4) Scaled-dot-product attention computed from Q, K, V, including why the √dₖ scaling is essential. (5) Padding masks vs causal masks and how they combine. (6) A defensible comparison of recurrent vs attention models on memory, parallelism, and long-range dependencies.

1 · A vanilla RNN, unrolled mathinteractive

A vanilla RNN reads one token at a time. At each step it combines the new input with the running hidden state:

ht = tanh( Wxh · xt + Whh · ht−1 + b )

The same weights Wxh and Whh are reused at every time step — weight sharing across time, the temporal analogue of weight sharing across space in a CNN.

Slide T to add more time steps. Each cell shows the recurrence applied with the same weights.

2 · Vanishing and exploding gradients in RNNs math

To compute ∂L/∂h₀, gradient back-propagates through every time step. By the chain rule, the contribution from step t picks up a product of t factors of Whh⊤ · diag(tanh′(zt)). If the largest singular value of that product is < 1 in expectation, the gradient vanishes. If it's > 1, it explodes. Either way, learning long-range dependencies becomes hopeless past about 10-20 steps.

The two standard fixes:

  • Gradient clipping. Cap the L2 norm of the gradient at some threshold; standard for any RNN training loop.
  • Gated architectures (LSTM, GRU). Add a separate cell state that's updated additively under gate control, bypassing the multiplicative chain.

3 · The LSTM cell: three gates mathinteractive

The LSTM maintains a hidden state h and a separate cell state c. Three gates — each a small sigmoid network — decide what to do at each step:

ft = σ(Wf · [ht−1, xt] + bf)     (forget)
it = σ(Wi · [ht−1, xt] + bi)     (input)
ĉt = tanh(Wc · [ht−1, xt] + bc)     (candidate)
ct = ft ⊙ ct−1 + it ⊙ ĉt     (new cell state)
ot = σ(Wo · [ht−1, xt] + bo)     (output)
ht = ot ⊙ tanh(ct)

Read the cell state update ct = f ⊙ ct−1 + i ⊙ ĉ as: forget some of what you knew, write some new information. Because the update is mostly additive (you can pass through f ≈ 1, i ≈ 0), gradients can travel a long way back in time.

4 · Cell state flowing through time interactive

One more demo to drive home the point: if the forget gate is wide open (close to 1) and the input gate is mostly closed at "boring" time steps, information from a key time step in the past survives for many steps. That is exactly what allows LSTMs to learn dependencies across hundreds of tokens.

5 · GRU as a simpler LSTM math

The Gated Recurrent Unit merges the cell and hidden states, and uses just two gates (reset and update):

rt = σ(Wr[ht−1, xt])    (reset)
zt = σ(Wz[ht−1, xt])    (update)
ĥt = tanh(Wh[rt ⊙ ht−1, xt])
ht = (1 − zt) ⊙ ht−1 + zt ⊙ ĥt

Three things to remember:

  • GRUs have ≈ 25 % fewer parameters than equivalent LSTMs.
  • On most NLP-ish tasks, GRU and LSTM perform very similarly. Pick by parameter budget and convention.
  • The same gradient-flow story applies: gates control how much past info passes through unchanged.

6 · Bidirectional LSTMs math

A bidirectional LSTM runs two LSTMs over the same sequence — one left to right, one right to left — and concatenates their hidden states at each position:

htBiLSTM = concat( ht→ , ht← )

This doubles the hidden dimension at each step but lets each position see both directions of context. The cost: it can no longer be used for purely autoregressive prediction, because the right-to-left pass cheats by looking at the future.

bilstm = nn.LSTM(input_size=D, hidden_size=H, batch_first=True,
                 bidirectional=True)
# out shape: (B, T, 2*H) — concatenation across directions
out, (h_n, c_n) = bilstm(x)

7 · Tokenization basics interactive

Before a model sees text, the text must become integer IDs. The three popular schemes:

  • Word-level. One ID per whitespace token. Big vocab; OOV problem.
  • Character-level. One ID per character. Tiny vocab; long sequences.
  • Subword (BPE, WordPiece, SentencePiece). Common patterns merged greedily. Modern default.

For BERT, "tokenization" actually means WordPiece: words become one or more subword pieces, with the special prefix ## marking "continuation":

"unbelievable" → ["un", "##believ", "##able"]    # WordPiece
"hello!"       → ["hello", "!"]
"COVID-19"     → ["co", "##vid", "-", "19"]

8 · Embedding lookup math

Once you have token IDs, you turn each into a vector with an embedding lookup. An embedding layer is just a learned matrix of shape (vocab_size, embedding_dim):

emb = nn.Embedding(num_embeddings=30000, embedding_dim=128)
ids = torch.tensor([[5, 22, 17, 0]])    # shape (B, T) = (1, 4)
x = emb(ids)                              # shape (1, 4, 128)
# parameters: 30000 * 128 = 3,840,000

That is a literal index-into-table lookup: the result for token 5 is row 5 of the matrix. Embeddings are trained jointly with the rest of the model — the geometry of the embedding space (which words are close to which) is what the network learns.

9 · Building Q, K, V mathinteractive

Attention computes a weighted average of value vectors, where the weights come from comparing queries to keys. Mechanically:

Q = X · WQ, K = X · WK, V = X · WV

Here X is the input sequence of shape (T, d_model). Each of WQ, WK, WV is a learned weight matrix of shape (d_model, d_k). After projection, Q, K, V all have shape (T, d_k).

The same input sequence X produces all three. This is "self-attention": queries, keys, and values all come from the same source. Cross-attention (Topic 5) uses different sources for the query vs. the key/value.

10 · Scaled-dot-product attention, step by step mathinteractive

The complete formula:

Attn(Q, K, V) = softmax( Q · K⊤ / √dₖ ) · V

Read it as a recipe with four steps:

  1. Score. S = Q · Kᵀ — for every query/key pair, dot product. Shape: (T_q, T_k).
  2. Scale. Divide by √dₖ to keep variance under control.
  3. Normalize. Row-wise softmax. Each row now sums to 1 — these are the attention weights.
  4. Mix. Multiply by V. Each row of the output is a weighted average of value rows.

11 · Why divide by √dₖ? mathinteractive

If q and k are random vectors of dimension dₖ with iid mean-zero variance-one components, then the variance of q · k is dₖ. So as dₖ grows, the magnitude of the dot product grows like √dₖ. Plug big numbers into softmax and one weight goes to ≈ 1 while the rest go to ≈ 0 — and the gradient of softmax in that regime is essentially zero.

Var( q · k ) = dₖ ⇒ dividing by √dₖ keeps Var = 1

Demo · Softmax with and without the √dₖ scaling, across dₖ

Top row: unscaled softmax of three random scores at various dₖ — saturates as dₖ grows. Bottom row: scaled by √dₖ — stays well-spread.

12 · Self-attention vs cross-attention math

Q fromK, V fromWhere
Self-attentionXX (same source)BERT encoder layers, GPT decoder layers (with causal mask)
Cross-attentiondecoder hidden stateencoder outputT5/BART decoder layers — grounds generation in the source

Mechanically the formula is the same; the difference is where Q vs. K, V come from. Cross-attention is what makes encoder–decoder transformers natural for translation, summarization, and structured generation: every decoder position can directly attend to every input position.

13 · Padding masks and causal masks interactive

Two masks you will use constantly:

  • Padding mask. Variable-length sequences are padded to the max length in the batch. The mask sets the scores at padded key positions to −∞ so softmax assigns zero weight.
  • Causal mask. Used in decoder-only transformers. Sets the score to −∞ at every (query, key) pair where key > query — no position can attend to the future.

You can combine them by element-wise OR-ing the two boolean masks before applying the −∞.

Set the per-sequence length (positions beyond it are padded). Toggle causal masking on/off to see how the two masks compose.

14 · Multi-head attention preview math

Single-head attention computes one (T, d_k) output per layer. Multi-head splits d_model into H heads of dimension d_k = d_model / H, runs scaled-dot-product attention independently in each head, then concatenates the H outputs and projects back to d_model:

MultiHead(Q, K, V) = Concat(head1, …, headH) · WO

Each head can specialize in a different sub-pattern (positional, syntactic, lexical). We'll dig into multi-head, encoder block, and positional encodings in Topic 5.

15 · LSTM vs attention — the trade-off interactive

Two things matter for the compute story:

  • Parallelism. An LSTM has a serial dependency in time. Self-attention processes all positions in parallel.
  • Memory in sequence length. Self-attention's score matrix is O(n²·d); a vanilla LSTM is O(n·d²). Past some threshold, the quadratic dominates.

16 · Attention weights as interpretability interactive

One free benefit of attention: the weights are directly interpretable. The weight at row i, column j tells you how much position i attended to position j in the input. Visualize as a heatmap and you have a per-prediction explanation.

Caveat. Attention weights are not a perfect explanation. The same model can give similar predictions with quite different weight patterns; you should treat heatmaps as an aid, not as a proof. But they are inexpensive, surface-level, and often diagnostic — especially when a model is wrong for an interesting reason.
