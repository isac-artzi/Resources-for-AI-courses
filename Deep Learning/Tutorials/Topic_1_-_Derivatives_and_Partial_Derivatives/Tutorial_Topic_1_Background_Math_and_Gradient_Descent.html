1 · Calculus quick-start mathinteractive

A neural network is just a long composition of differentiable functions. So a deep-learning engineer's relationship with calculus is mostly mechanical: you need to know how to differentiate the standard library of functions, and you need fluency with the chain rule for compositions and the sum/product rules for combinations.

1.1 · The single-variable derivative

For a function f: ℝ → ℝ, the derivative at x is the slope of the tangent line:

f′(x) = limh→0 ( f(x + h) − f(x) ) / h

You will rarely compute a derivative from this definition. Instead you'll lean on a handful of rules:

RuleStatementExample
Powerd/dx (xn) = n · xn−1d/dx (x³) = 3x²
Exponentiald/dx (ex) = exd/dx (e2x) = 2 · e2x (with chain)
Logd/dx (ln x) = 1/xd/dx (ln(x² + 1)) = 2x / (x² + 1)
Sumd/dx (f + g) = f′ + g′d/dx (x³ + 2x) = 3x² + 2
Productd/dx (f · g) = f′·g + f·g′d/dx (x · sin x) = sin x + x · cos x
Chaind/dx f(g(x)) = f′(g(x)) · g′(x)d/dx sin(x²) = 2x · cos(x²)

1.2 · Partial derivatives

For a function of multiple variables, say f(x, y), the partial with respect to x is the derivative you get by treating y as a constant. Notation:

∂f/∂x = limh→0 ( f(x + h, y) − f(x, y) ) / h

The gradient is the vector of partials. For f(w, b) = (w · x + b − y)² (one training example):

∇f = ( ∂f/∂w , ∂f/∂b ) = ( 2(w·x + b − y) · x , 2(w·x + b − y) )

That gradient is exactly what gradient descent will use to update w and b at each step.

Demo · A 2-D function and its gradient field

The contour plot below is f(w, b) = (w − 2)² + (b + 1)² — a paraboloid centered at (2, −1). The arrows are the gradient direction at each grid point; gradient descent moves opposite to those arrows, straight toward the minimum.

2 · The chain rule, visualized mathinteractive

The chain rule says: to differentiate a composition f(g(x)), differentiate the outer at the value of the inner, then multiply by the derivative of the inner. It is the single most-used rule in deep learning because every layer of a neural network is exactly such a composition.

d/dx [ f(g(x)) ] = f′(g(x)) · g′(x)

Why this is the engine of backpropagation: the loss is L( fN( fN−1( … f1(x) ) ) ). Repeatedly applying the chain rule from the outside in gives a product of local Jacobians — one per layer — that you can compute in a single backward pass.

Demo · Three-step chain rule walkthrough

Take f(x) = sin(x²). Set g(x) = x² and h(u) = sin(u); then f(x) = h(g(x)). The chain rule says f′(x) = h′(g(x)) · g′(x) = cos(x²) · 2x. The plot below shows all three pieces in sync as you slide x.

3 · Backpropagation: the chain rule on a tiny network mathinteractive

Consider the smallest possible network: one input, one weight, one bias, one activation, one squared-error loss.

z = w · x + b, a = σ(z), L = ½ ( a − y )²

To train, we need ∂L/∂w and ∂L/∂b. By the chain rule:

∂L/∂w = (a − y) · σ′(z) · x,     ∂L/∂b = (a − y) · σ′(z)

Read this as: the loss propagates the residual (a − y) through the activation gradient σ′(z), and the input x picks up the multiplier for the weight gradient. The same pattern, layer by layer, is exactly what PyTorch's autograd computes for you.

4 · Gradient descent on a 1-D loss interactive



Gradient descent updates a parameter by stepping against the gradient:


w ← w − η · ∂L/∂w


Two facts you should never get wrong:


    
  
  • Gradient descent moves against the gradient. The gradient itself points uphill.
    • 
  
  • The learning rate η controls how big each step is. Too small ⇒ painfully slow. Too large ⇒ oscillation or divergence.
    •

5 · Learning-rate regimes interactive



For a quadratic loss L(w) = (w − c)², gradient descent updates obey wt+1 = wt − 2η(wt − c). The error et = wt − c evolves as et+1 = (1 − 2η) · et. Three regimes:


    
  
  • |1 − 2η| < 1 ⇒ converges monotonically (small η) or with damped oscillation (η between 0.5 and 1).
    • 
  
  • |1 − 2η| = 1 ⇒ flat oscillation forever; happens exactly at η = 1.
    • 
  
  • |1 − 2η| > 1 ⇒ diverges; happens for η > 1.
    • 



This is why you see "divide your learning rate by 10" as a debugging move; it is also why warmup schedules exist for transformers, where the loss surface is much more complicated.

7 · Batch, mini-batch, and stochastic gradient descent interactive



So far the "gradient" was the gradient of the loss on a single example. In practice you minimize an average loss over many examples:


L(θ) = (1/N) Σi=1..N Li(θ)


You can compute this gradient in three ways:


    
  
  • Batch gradient descent. Use all N examples per step. Lowest-variance gradient; expensive per step; smooth loss curve.
    • 
  
  • Stochastic gradient descent (SGD). Use a single random example per step. Cheap; very noisy gradient; loss curve looks like static.
    • 
  
  • Mini-batch SGD. Use a small batch (e.g., 32, 128). The standard. Compute-efficient on GPUs; gradient noise is a feature — it helps escape narrow minima.
    •

8 · Optimizers: SGD, momentum, and Adam interactive



Plain SGD takes a step in the direction of the current gradient. Two famous improvements:


    
  
  • Momentum. Keep an exponentially-decaying average of past gradients and step in that direction. Smooths zig-zags through long, narrow valleys.
    • 
  
  • Adam. Use both a moving average of the gradient (momentum) and a moving average of the squared gradient (adapts the learning rate per parameter). The default optimizer for most deep networks.
    • 



The update rules:


SGD            : w ← w − η · g
SGD + momentum : v ← μ · v − η · g ;  w ← w + v
Adam           : m ← β₁ · m + (1 − β₁) · g           # 1st-moment estimate
                 v ← β₂ · v + (1 − β₂) · g²          # 2nd-moment estimate (elementwise)
                 m̂ ← m / (1 − β₁ᵗ) ;  v̂ ← v / (1 − β₂ᵗ)   # bias correction
                 w ← w − η · m̂ / (√v̂ + ε)

9 · PyTorch autograd, end to end code



PyTorch tracks every operation on tensors that have requires_grad=True, building a dynamic computation graph at forward time. When you call .backward() on a scalar, it walks the graph backward and fills .grad on every leaf tensor with the partial derivative of that scalar with respect to the tensor.



import torch

x = torch.tensor(2.0, requires_grad=True)
y = torch.tensor(3.0, requires_grad=True)

f = x ** 2 + 3 * x * y + y ** 2     # forward
f.backward()                         # backward

print(x.grad.item())                 # ∂f/∂x = 2x + 3y = 4 + 9 = 13
print(y.grad.item())                 # ∂f/∂y = 3x + 2y = 6 + 6 = 12



Three things to remember:


    
  
  • One backward per scalar. f must be a scalar (a 0-D tensor) to call .backward() without arguments. Loss functions like nn.MSELoss always return a scalar.
    • 
  
  • Gradients accumulate. If you call .backward() twice without clearing, the second call adds to .grad. That is why every training loop calls optimizer.zero_grad() before .backward().
    • 
  
  • Detach to break the graph. If you do w = w - lr * g while w.requires_grad=True outside a with torch.no_grad(): block, you'll either get an autograd error or create a useless graph. Use the optimizer.
    •

11 · Linear regression: fit, evaluate, interpret interactive



A linear regression model predicts ŷ = w·x + b. With multiple features you have a vector of weights and the same single-line story: ŷ = w⊤x + b. Three evaluation metrics you should know cold:



  
MSE
mean((ŷ − y)²) — penalizes large errors quadratically

  
MAE
mean(|ŷ − y|) — robust to outliers, linear gradient magnitude

  
R²
1 − SS_res / SS_tot — scale-free; 1 is perfect fit, 0 is no better than predicting the mean

12 · Train / validation / test splits interactive



You can't honestly measure generalization on the same data you trained on. The standard recipe:


    
  
  • Training set — used to fit the parameters.
    • 
  
  • Validation set — used to pick hyperparameters (learning rate, model size, dropout).
    • 
  
  • Test set — used once, at the very end, to report an honest generalization number.
    • 



The cardinal sin is "validation leakage": tuning on the test set, looking at the test set during development, or accidentally training on validation. Once you've looked, you've burned the set.

13 · Reproducibility and random seeds code



If two runs of your code produce different metrics, you do not have a reproducible experiment, and you cannot run an honest ablation. Three sources of nondeterminism to control:


    
  
  • Python / NumPy. random.seed(s) and numpy.random.seed(s).
    • 
  
  • PyTorch. torch.manual_seed(s) for CPU; torch.cuda.manual_seed_all(s) for all GPUs.
    • 
  
  • CUDA non-determinism. Set torch.backends.cudnn.deterministic = True and accept a small slowdown.
    • 




import os, random, numpy as np, torch

def set_seed(s: int = 42):
    random.seed(s)
    np.random.seed(s)
    torch.manual_seed(s)
    torch.cuda.manual_seed_all(s)
    torch.backends.cudnn.deterministic = True
    torch.backends.cudnn.benchmark = False
    os.environ["PYTHONHASHSEED"] = str(s)




  Reproducibility ≠ stability. A reproducible run can still be a fragile run — same seed, same numbers, but tiny code changes (a fused op, a new GPU kernel) might shift them. The right move is to also run a small grid of seeds and report a mean + standard deviation.





14 · Convergence diagnostics interactive



When has training "converged"? Three signals you want to see together:


    
  
  • Training loss has flattened. Per-epoch slope is near zero for several epochs.
    • 
  
  • Validation loss has flattened or started rising. A widening train/val gap is the classic overfitting signature.
    • 
  
  • Gradient norm has stopped shrinking dramatically. If the gradient norm is still huge, you have not converged regardless of how flat the loss curve looks.
    •

17 · Pydantic schemas and input validation codeinteractive



FastAPI uses Pydantic models to validate request and response bodies. The contract is declared once and enforced automatically: bad inputs return a structured 422 Unprocessable Entity with a JSON pointer telling the client exactly which field failed and why.



from pydantic import BaseModel, Field

class TrainRequest(BaseModel):
    dataset_id: int
    lr:         float = Field(gt=0, le=1.0, default=0.01)
    epochs:     int   = Field(ge=1,  le=1000, default=100)
    batch_size: int   = Field(ge=1,  le=4096, default=32)

class TrainResponse(BaseModel):
    run_id: int
    mse:    float
    r2:     float
