1 · The convolution operation interactivemath

A 2-D convolution slides a small kernel K over an input feature map and at each position computes a dot product between the kernel and the local patch. The same kernel is used at every position — that is the weight sharing property that makes CNNs parameter-efficient.

out[i, j] = Σr=0..K-1 Σc=0..K-1 K[r, c] · in[i+r, j+c]

Three classical kernels you'll see in any textbook:

KernelEffect
Identity (1 in the center)Output = input
Edge (Laplacian, 8 in the center, −1 around)Highlights intensity changes
Box blur (1/9 everywhere)Local averaging
Sharpen (5 in the center, −1 cross)Boosts local contrast

2 · Output-size arithmetic math

For a square input of side I, square kernel K, stride S, and padding P:

O = ⌊ (I − K + 2P) / S ⌋ + 1

Three configurations you should memorize because they show up everywhere:

LayerIKSPO
"same" 3×3 conv3231132 (preserves size)
Strided 3×3 down-sample3232116 (halves size)
"valid" 7×7 (no padding)3271026

4 · Parameters per conv layer mathinteractive


For a convolutional layer with C_in input channels, C_out output channels, and square kernel K:


params = C_out · ( C_in · K · K + 1 )


The +1 is the per-output-channel bias. Crucially, the parameter count does not depend on spatial size — the same kernel is shared across every spatial position. That is why CNNs scale to 224×224 inputs without exploding parameters.




  
Demo · Compare conv layer to a fully connected layer on the same input

  
For input (B, C_in, H, W), a fully connected layer that reads all pixels would have (C_in · H · W) · (C_out · H · W) weights — astronomical. The conv layer reuses the same small kernel across every position.

  

    

    

    

    

    

  

  






5 · Multi-channel convolutions math


Real images have multiple input channels (e.g., 3 for RGB). A multi-channel kernel is actually a small 3-D tensor of shape (C_in, K, K); you have C_out of them stacked, so the weight tensor is (C_out, C_in, K, K).


At each spatial position, the output for channel j is the sum across all input channels of the channel-wise dot products:


out[j, i, k] = Σc=0..C_in-1 Σr, s W[j, c, r, s] · in[c, i+r, k+s]



# PyTorch
conv = nn.Conv2d(in_channels=3, out_channels=64, kernel_size=3, padding=1)
# weight shape: (64, 3, 3, 3) — i.e., 64 different 3×3×3 mini-volumes
# bias shape:   (64,)
print(sum(p.numel() for p in conv.parameters()))    # 64*3*3*3 + 64 = 1792




6 · Padding modes interactive


Padding controls what to put at the borders so the kernel can "reach" edge pixels. Three common choices:


    
  
  • Zero padding (the PyTorch default). Pads with 0s; introduces a small boundary artifact.
    • 
  
  • Reflection padding. Mirrors the image at the boundary. Common for image-restoration tasks.
    • 
  
  • Replication padding. Extends edge pixels.

7 · Receptive field of stacked kernels interactive


The receptive field of a neuron is the spatial region of the input that affects it. For stride-1 stacks of K×K convolutions:


RFn = 1 + n · (K − 1)


So three stacked 3×3 convolutions reach a 7×7 region; five stacked 3×3 convolutions reach 11×11. This is why "small kernels, deeper stack" became the design pattern: same receptive field as a single 7×7 with ~55% fewer parameters and three non-linearities instead of one.

9 · Dilated convolutions math


Dilation inserts gaps between kernel positions. A 3×3 conv with dilation d has the same parameter count as a regular 3×3 conv but covers a 1 + 2·d region. This was popularized by WaveNet (1-D dilations) and DeepLab (semantic segmentation):



  
Dilation dReceptive field of a 3×3 kernel

  
1 (no dilation)3 × 3

  
25 × 5

  
49 × 9

  
817 × 17




Exponentially-growing dilation across layers gives an enormous receptive field without sub-sampling.




10 · Residual blocks mathinteractive


A residual (skip) connection adds the input of a block to its output:


y = ReLU( x + F(x) )


where F is the block's main path (usually two convs with batch norm). Three reasons this is the single most important architectural idea in modern CNNs:


    
  
  • Identity is free. If F(x) = 0, the block reduces to ReLU(x) — so the model can choose to do nothing on this block. Deep networks are not forced to compute through every layer.
    • 
  
  • Gradients flow. The additive path lets gradient flow back through the skip even when F's gradient is small. This is what made networks like ResNet-152 trainable in the first place.
    • 
  
  • Empirical accuracy. Deep residual networks consistently outperform plain networks of equivalent depth.
    •

11 · Batch normalization in CNNs math


BatchNorm in 2-D normalizes per channel across the batch and the spatial dimensions:


μ_c = mean over (B, H, W) of x[:, c, :, :],  σ²_c = variance over same;
  out = γ_c · (x − μ_c) / √(σ²_c + ε) + β_c


The two parameters per channel γ_c and β_c are learned; μ_c and σ²_c use a running average at inference (which is what model.eval() turns on). BN typically goes right after Conv2d and before ReLU:


nn.Sequential(
    nn.Conv2d(64, 64, 3, padding=1),
    nn.BatchNorm2d(64),     # 128 learned params: 64 γ's + 64 β's
    nn.ReLU(inplace=True),
)




12 · Data augmentation interactive


Augmentation expands the training set by applying label-preserving transformations. For natural images, the standard CIFAR-10 recipe is:


transforms.Compose([
    transforms.RandomCrop(32, padding=4),    # crop after padding
    transforms.RandomHorizontalFlip(),
    transforms.ToTensor(),
    transforms.Normalize(mean=[0.49, 0.48, 0.44],
                         std =[0.25, 0.24, 0.26]),
])

13 · Learning-rate scheduling interactive


A constant LR for the whole training run is rarely optimal. Three common schedules:


    
  
  • Step decay. Multiply LR by 0.1 at fixed milestones (e.g., epoch 30, 60).
    • 
  
  • Cosine annealing. LR follows a half-cosine from η_max down to (approximately) 0 across the training horizon. Standard for image classifiers.
    • 
  
  • Cosine with warmup. Linearly ramp from a tiny LR up to η_max for the first ~5% of training, then anneal as cosine. Required for transformers.
    •

14 · Transfer learning math


When you have a small dataset, training from scratch is the wrong move. Transfer learning loads a backbone pre-trained on ImageNet (say, ResNet-50) and fine-tunes it on your target task. Two standard variants:



  
VariantWhat you changeWhen to use

  
Linear probeFreeze all backbone weights; train a fresh classifier head only.Tiny dataset (< 1k images); want fast iteration.

  
Fine-tune the whole backboneTrain every weight at a very small LR (e.g., 1e-4 to 1e-5).Moderate dataset (10k+ images); want maximum accuracy.




from torchvision.models import resnet50, ResNet50_Weights

m = resnet50(weights=ResNet50_Weights.IMAGENET1K_V2)
for p in m.parameters(): p.requires_grad = False          # freeze
m.fc = nn.Linear(m.fc.in_features, num_classes)            # fresh head
# only fc parameters will receive gradients




15 · Grad-CAM: what is the model looking at? interactivemath


Grad-CAM produces a class-discriminative saliency map by weighting the feature maps of a target conv layer by the global-average-pooled gradients of a chosen class score with respect to those feature maps:


L_GradCAM(c) = ReLU( Σk α(c)k · Ak ),  α(c)k = meani, j ∂yc / ∂Aki, j


Useful diagnostic: if the saliency highlights a watermark, background, or other shortcut feature, you have a dataset failure, not a model failure.
