2 · Tokenizer output, in detail codeinteractive

Every Hugging Face tokenizer call returns a dict-like object with these keys, all PyTorch tensors:

  • input_ids — the token IDs for each input. Shape (B, T_max).
  • attention_mask — 1 for real tokens, 0 for padding. Shape (B, T_max). Passed to the model so attention zeros out padded positions.
  • token_type_ids (some models) — 0/1 indicating segment A vs B for sentence-pair tasks.

3 · From pre-training to fine-tuning math


Pre-training is the expensive part: train a transformer on hundreds of billions of tokens with a self-supervised objective, optimizing all parameters from scratch. Fine-tuning is the cheap part: start from the pre-trained weights, attach a small task-specific head, and continue training on your labeled data.


The key insight: pre-trained features already work on your task. You just need small, gentle updates to specialize them. That is why fine-tuning uses learning rates like 2e-5 instead of the 1e-3 typical for training from scratch.




  
Pre-trainingFine-tuning

  
Dataset sizeHundreds of billions of tokensHundreds to thousands of labeled examples

  
ComputeDays on hundreds of GPUsMinutes to hours on one GPU (or Colab free tier)

  
Learning rate1e-3 to 1e-4 with warmup2e-5 to 5e-5 with short warmup

  
Epochs1-2 over the corpus2-5 over the fine-tuning set

  
LossSelf-supervised (MLM, CLM, …)Task-specific (cross-entropy on labels)

5 · The Hugging Face Trainer loop code


The Trainer owns the loop so you don't have to write it. The minimal recipe:


from transformers import (AutoTokenizer, AutoModelForSequenceClassification,
                          TrainingArguments, Trainer)
import evaluate

tokenizer = AutoTokenizer.from_pretrained("distilbert-base-uncased")
model     = AutoModelForSequenceClassification.from_pretrained(
    "distilbert-base-uncased", num_labels=2)

def tokenize(batch):
    return tokenizer(batch["text"], padding="max_length", truncation=True, max_length=256)

ds = ds.map(tokenize, batched=True)

f1_metric = evaluate.load("f1")
def compute_metrics(eval_pred):
    logits, labels = eval_pred
    preds = np.argmax(logits, axis=-1)
    return f1_metric.compute(predictions=preds, references=labels, average="binary")

args = TrainingArguments(
    output_dir              = "out",
    num_train_epochs        = 3,
    per_device_train_batch_size = 16,
    per_device_eval_batch_size  = 64,
    learning_rate           = 2e-5,
    warmup_ratio            = 0.06,
    weight_decay            = 0.01,
    eval_strategy           = "epoch",
    save_strategy           = "epoch",
    load_best_model_at_end  = True,
    metric_for_best_model   = "f1",
    greater_is_better       = True,
)

trainer = Trainer(model=model, args=args,
    train_dataset=ds["train"], eval_dataset=ds["test"],
    compute_metrics=compute_metrics, tokenizer=tokenizer)

trainer.train()
trainer.save_model("models/spam-classifier-v1")



What the Trainer does at each step:


    
  
  1. Pull a mini-batch from the train DataLoader.
    2. 
  
  2. Tokenize (if not already done), move to device, run the forward pass.
    3. 
  
  3. Compute the loss (returned by the model when labels are passed).
    4. 
  
  4. Call loss.backward(), clip gradients, step the optimizer, step the scheduler.
    5. 
  
  5. If on an eval step, switch to eval mode, run over the eval DataLoader, call compute_metrics, log results.
    6. 
  
  6. Save the best checkpoint according to metric_for_best_model.
    7. 





6 · Cross-entropy, focal loss, label smoothing math


For most classification fine-tuning, cross-entropy is what you want. Two variants you should know:


    
  
  • Label smoothing. Replace one-hot targets with (1 − ε) · one_hot + ε / C. Discourages the model from being arbitrarily confident; improves calibration. Standard in transformer training (ε ≈ 0.1).
    • 
  
  • Focal loss. Multiply CE by (1 − pₜ)γ; down-weights easy examples and up-weights hard ones. Useful for severely imbalanced classification (e.g., detection).
    • 



L_CE         = −log p_y                                    # standard
L_smooth     = −(1-ε) · log p_y − (ε/C) · Σⱼ log pⱼ        # label smoothing
L_focal      = −(1 − p_y)^γ · log p_y                       # focal





7 · Precision, recall, F1 — worked mathinteractive


For binary spam classification with TP = 84, FN = 16, FP = 10, TN = 90:


precision = TP / (TP + FP) = 84 / (84 + 10) = 84 / 94  = 0.894
recall    = TP / (TP + FN) = 84 / (84 + 16) = 84 / 100 = 0.840
F1        = 2 · P · R / (P + R) = 2 · 0.894 · 0.840 / (0.894 + 0.840)
          ≈ 0.866

8 · ROUGE-1 and ROUGE-L mathinteractive


For summarization, ROUGE compares the model's summary against the reference.


    
  
  • ROUGE-1: token-level F1 of unigram overlap.
    • 
  
  • ROUGE-2: token-level F1 of bigram overlap (consecutive token pairs).
    • 
  
  • ROUGE-L: F1 based on the length of the longest common subsequence (LCS) — captures fluency and order without requiring strict contiguity.

9 · Perplexity math


Perplexity is the exponential of the average per-token negative log-likelihood:


PPL = exp( − (1/N) · Σᵢ log pᵢ(yᵢ) )


Interpretation: "the effective vocabulary size the model is uncertain over at each step." A perplexity of 1 means perfect prediction; perplexity equal to V (vocab size) means random guessing. Strong language models on web text typically have perplexity in the 10-30 range.




  
PPLReading

  
1Perfect: model assigns probability 1 to every next token

  
30Strong large model on web text

  
100Small model or out-of-distribution text

  
30 000Random guessing on a 30 K vocab






10 · 95 % bootstrap confidence intervals interactive


A single F1 number from one test set is uncertain. The bootstrap estimates that uncertainty by resampling the test set with replacement many times and recomputing F1 on each resample. The 2.5th and 97.5th percentiles form the 95 % CI.

13 · LoRA and parameter-efficient fine-tuning math


Full fine-tuning updates every weight in a 7-billion-parameter model — feasible only with a lot of GPU memory. Parameter-efficient fine-tuning (PEFT) freezes the original weights and adds a tiny set of trainable parameters elsewhere.


The most popular flavor is LoRA (Low-Rank Adaptation). For a frozen weight matrix W of shape (d, d), LoRA adds ΔW = B · A with B: (d, r) and A: (r, d) for some small rank r (8, 16, 64). At inference, the forward pass becomes:


x → x · W + x · B · A    (instead of just x · W)


The trainable parameters drop from d² per matrix to 2 · d · r. For d = 4096, r = 8 that is ≈ 16 M → 65 K — a 250× reduction. LoRA reaches ≈ 95-99% of full fine-tuning quality on most tasks at a tiny fraction of the cost.
