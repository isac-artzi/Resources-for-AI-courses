1 · The perceptron model math

The simplest neural unit. Given an input vector x and weights w with a bias b:

z = w · x + b, ŷ = step(z) = { 1 if z ≥ 0, 0 otherwise }

This is just a hyperplane separator. Geometrically, the weight vector w is the normal of the decision boundary, the bias b shifts it away from the origin, and points with positive z are on the positive side. Training originally used a "perceptron update rule": when a prediction is wrong, push w in the direction of the misclassified example. With modern losses and gradient descent (Topic 1), the picture is the same — just smoother.

What a single perceptron cannot learn

A single perceptron defines a single linear boundary. Anything you cannot separate with one straight line is out of reach. The canonical counter-example is XOR.

4 · Matrix-form forward propagation math

For a single input vector x of dimension n_in, a weight matrix W of shape (n_in, n_hidden), a bias vector b of shape (n_hidden,), and an elementwise activation σ:

h = σ( W⊤ · x + b )

For a batch X of shape (B, n_in) — which is how PyTorch stores data — the formula is even cleaner because everything broadcasts:

H = σ( X · W + b )

That X · W has shape (B, n_hidden); the bias broadcasts across the batch dimension; the activation is applied elementwise. You should know the shape of every intermediate tensor in any network you read.

A worked two-layer pass

x:   (4, 10)            # batch of 4 examples, 10 features each
W1:  (10, 64)
b1:  (64,)
h1 = ReLU(x @ W1 + b1)  # shape (4, 64)
W2:  (64, 1)
b2:  (1,)
out = h1 @ W2 + b2      # shape (4, 1)  -> 4 raw logits

6 · Activations and their derivatives interactivemath


Activation functions are the source of non-linearity. The shape of an activation matters; the shape of its derivative matters even more — because that derivative is exactly what backprop multiplies through.



  
Activationσ(z)σ′(z)Typical use

  
Sigmoid1 / (1 + e−z)σ(z) · (1 − σ(z))Binary output, gated mechanisms (e.g. LSTM gates)

  
tanh(ez − e−z) / (ez + e−z)1 − tanh²(z)Zero-centered hidden activation in older nets

  
ReLUmax(0, z)1 if z > 0 else 0Default for CNNs and MLPs

  
GELU0.5 · z · (1 + tanh(√(2/π) · (z + 0.044715 z³)))smooth, non-monotonicDefault in transformers






  
Demo · σ and σ′ side by side

  
Notice how sigmoid and tanh both saturate: outside about ±3, the derivative is essentially zero. That is the source of the "vanishing gradient" problem in deep stacks (next section). ReLU's derivative is 0 or 1 — no saturation in the active region.

  





7 · Backpropagation through a small MLP mathinteractive


A two-layer MLP is the smallest model you can actually backprop by hand:



  z₁ = W₁ · x + b₁,  a₁ = σ(z₁),  z₂ = W₂ · a₁ + b₂,  ŷ = σ(z₂),  L = ½(ŷ − y)²



Applying the chain rule outward-in:


∂L/∂ŷ   = (ŷ − y)
∂L/∂z₂  = ∂L/∂ŷ  ·  σ'(z₂)
∂L/∂W₂  = ∂L/∂z₂ · a₁ᵀ        # shape matches W₂
∂L/∂b₂  = ∂L/∂z₂

∂L/∂a₁  = W₂ᵀ · ∂L/∂z₂          # backpropagated through W₂
∂L/∂z₁  = ∂L/∂a₁ ⊙ σ'(z₁)       # ⊙ = elementwise multiply
∂L/∂W₁  = ∂L/∂z₁ · xᵀ
∂L/∂b₁  = ∂L/∂z₁


Every layer takes the gradient coming back from above, multiplies by the local activation derivative, and uses the input to that layer to compute the weight gradient. PyTorch's autograd does this for you, but you should know the pattern so you can debug shapes when something inevitably goes wrong.

8 · Vanishing and exploding gradients interactive


When you stack many layers with saturating activations, the gradient that reaches the early layers is a product of many small numbers (vanishing), or many large numbers (exploding). Symptoms and fixes:



  
PathologySymptomFixes

  
Vanishing gradientsEarly-layer weights barely change across many epochs while later-layer weights update normally; loss plateaus high.ReLU/GELU activations, He/Xavier initialization, batch / layer norm, residual connections.

  
Exploding gradientsLoss spikes to NaN or to huge values; parameters become inf in a few iterations.Gradient clipping (nn.utils.clip_grad_norm_), smaller learning rate, better init.

Saturating activations and bad init combine to make the early-layer gradient effectively zero.

  

    

    

      
      
    

    

      
      
    

    

  

  





9 · Weight initialization: zero, normal, Xavier, He interactive


Bad init is one of the most common reasons a network "won't train." Four standard schemes:


    
  
  • All zeros. All hidden units learn the same thing — broken symmetry; the network reduces to a single linear unit. Never use for hidden layers.
    • 
  
  • Random normal (large variance). Activations saturate fast; gradients explode.
    • 
  
  • Xavier (Glorot). var(W) = 2 / (n_in + n_out). Designed for sigmoid/tanh.
    • 
  
  • He (Kaiming). var(W) = 2 / n_in. Designed for ReLU; the standard for modern ReLU/GELU nets.
    • 





  
Demo · Activation distribution at every layer

  
Forward-propagate a Gaussian input through a 6-layer ReLU stack initialized four different ways. The histogram for each layer shows the distribution of activations. Healthy init: roughly constant variance across layers.

  

    
    
    
    
    

  

  





10 · Dropout, visualized interactive


Dropout randomly zeroes a fraction p of hidden units during training (and rescales the survivors by 1/(1−p) to keep the expected activation the same). It prevents co-adaptation: the network cannot rely on any single unit always being there, so units learn more robust features. At inference time dropout is turned off (model.eval() handles this).



  
Demo · A single hidden layer under dropout

  
Each column is a forward pass through a 1×12 hidden layer. Press Step to draw a new dropout mask. Notice how the same input produces different active subnetworks each step — at inference, all 12 units are always on.

  

    

    
    
    

  

  





11 · Batch normalization, conceptually math


BatchNorm normalizes activations along the batch dimension at each layer:



  ẑ = (z − μ_B) / √(σ²_B + ε),  out = γ · ẑ + β



Here μ_B and σ²_B are the per-channel mean and variance of the batch; γ and β are learned scale and shift. At inference time, the running averages of μ_B and σ²_B collected during training are used — that is what model.eval() activates.


The practical effects:


    
  
  • Stabilizes the distribution of layer inputs, so deeper stacks train without dying.
    • 
  
  • Allows higher learning rates than the same network without BN.
    • 
  
  • Acts as a mild regularizer because of the batch-dependent normalization.
    • 




  Watch the batch size. Very small batches give noisy μ_B and σ²_B estimates, which can hurt training. Inference with batch = 1 works fine because BatchNorm uses the running averages.





12 · Loss for classification: BCE, cross-entropy, softmax mathcode


For classification you almost always use a probabilistic loss. Three patterns you must recognize:


Binary classification


Output one logit z; apply sigmoid; use BCE.


L_BCE = − [ y · log σ(z) + (1 − y) · log(1 − σ(z)) ]


In PyTorch: nn.BCEWithLogitsLoss() (it applies the sigmoid inside, in a numerically-stable way).


Multi-class classification


Output a logit per class; apply softmax; use cross-entropy.


softmax(z)i = exp(zi) / Σj exp(zj),  L_CE = − log softmax(z)y


In PyTorch: nn.CrossEntropyLoss() takes raw logits (not soft-maxed) and integer class indices.



  Do not double-softmax. Don't apply nn.Softmax to your logits before passing them to nn.CrossEntropyLoss — the loss applies log-softmax internally. Doing both shrinks the gradient and slows learning.





13 · Confusion matrix → precision, recall, F1 interactive


For a binary classifier the four cells of the confusion matrix are:



  
predicted positivepredicted negative

  
true positive classTPFN

  
true negative classFPTN




Derived metrics:



  precision = TP / (TP + FP),   recall = TP / (TP + FN),  F1 = 2 · P · R / (P + R)



Decision-threshold-dependent metrics:



  FPR = FP / (FP + TN),   TPR = TP / (TP + FN) = recall

14 · ROC curve and AUC interactive


Threshold a classifier's predicted probabilities at every possible value and plot (FPR, TPR) — that's the ROC curve. The area under it (AUC) is a single number summary; AUC = 1 is a perfect classifier, AUC = 0.5 is random guessing.


AUC is also threshold-free — it does not depend on where you set the decision boundary. That is why teams report it alongside the threshold-dependent confusion matrix.

15 · Calibration plots interactive


A calibration plot bins predictions by predicted probability and plots, for each bin, the empirical positive rate. A perfectly calibrated classifier lies on the diagonal: when it says "70%", the true positive rate among those predictions is also 70%.


Most deep networks are over-confident — their predicted probabilities are too extreme. Temperature scaling (dividing logits by a scalar before softmax) is the standard cheap fix.

16 · scikit-learn Pipeline structure code


For tabular data the preprocessing must travel with the model. The standard pattern combines a ColumnTransformer (different transforms for numeric and categorical columns) with a model in a single Pipeline:


num_cols = ["age", "education_num", "hours_per_week"]
cat_cols = ["workclass", "occupation", "sex"]

prep = ColumnTransformer([
    ("num", StandardScaler(),                   num_cols),
    ("cat", OneHotEncoder(handle_unknown="ignore"), cat_cols),
])

pipe = Pipeline([
    ("prep", prep),
    ("clf",  LogisticRegression(max_iter=1000)),
])

pipe.fit(X_train, y_train)
joblib.dump(pipe, "model.joblib")    # the WHOLE pipeline, including the fitted scaler


Three benefits, all of which the product mindset cares about:


    
  
  • No train/serve skew. Inference uses the exact same preprocessing as training.
    • 
  
  • One artifact. A single pickle/joblib file ships the whole transformation chain.
    • 
  
  • Cleaner cross-validation. CV folds re-fit the scaler on each training partition, so test-set statistics never leak.
    •
