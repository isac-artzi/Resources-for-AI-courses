# Model Card — Regress-It

## Model details
- **Type:** 1-D linear regression, `y = w·x + b`, fit with mini-batch SGD (PyTorch `nn.Linear`).
- **Training:** per-run hyperparameters (learning rate, batch size, epochs) supplied by the API request.
- **Owner:** your team (add your names).

## Intended use
- **Primary:** an interactive teaching demo showing how a linear model trains and how learning rate affects convergence vs. divergence.
- **Out of scope:** any real decision-making. This model is trained on *synthetic* data with a known generating line.

## Data
- Synthetic points sampled from `y = slope·x + intercept + N(0, noise²)`.
- Generated by `shared/data.generate_linear` and stored in the Supabase `datasets` table.

## Metrics
- Reported on a held-out 20% test split: **MSE**, **MAE**, **R²**.
- Every run's metrics are persisted to the Supabase `runs` table (single source of truth — not recomputed in the UI).

## Limitations
- Assumes a linear relationship; will underfit any non-linear signal.
- Sensitive to learning rate: too large diverges, too small under-trains within the epoch budget.
- No uncertainty estimate is returned with a prediction.

## Ethical considerations
- A non-technical client must be told the model's limits plainly (honesty/stewardship): it is a demonstration on synthetic data, not a validated predictor. Disclose the assumptions and the held-out error, and do not overstate confidence.

## What we log (and don't)
- We log run hyperparameters, metrics, fitted weights, and each prediction `(run_id, x, yhat)`.
- We do **not** log any personal data — the dataset is synthetic by construction.
