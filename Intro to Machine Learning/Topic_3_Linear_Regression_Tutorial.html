Topic 3 · Linear Regression with pandas & scikit-learn

Topic 3 is your first machine-learning model. You will fit a Linear Regression in scikit-learn, judge it with R² and RMSE on a held-out test set, and present its prediction with an honest uncertainty range. This page builds your intuition with five interactive panels — drag, click, and watch the math come alive.

Scope at the introductory level: LinearRegression, R², RMSE, chronological train/test split, residual histogram, and a simple uncertainty range built from the residual standard deviation. Ridge / Lasso / adjusted R² / Q-Q plots / formal prediction intervals are out of scope.

0. What is regression, in one sentence?

Regression means predicting a continuous number from one or more features. Predicting tomorrow's closing price is regression. Predicting a house price from square footage and bedrooms is regression. Predicting whether an email is spam is not regression — that is classification, which is Topic 5.

1. What Linear Regression actually fits

A linear regression with one feature is just a line: ŷ = b₀ + b₁·x. Given training data, scikit-learn picks the line that makes the sum of squared residuals as small as possible.

Below is a small dataset. Drag the sliders to change the line by hand. The pink vertical bars are the residuals (observed − predicted). The numbers update in real time. The button "Best fit" snaps the sliders to the line that minimizes the residual squared sum — that is what LinearRegression().fit() does for you.

Tip: R² = 1 means the line passes through every point. R² near 0 means the line is no better than predicting the mean. R² CAN be negative on test data when the line is worse than predicting the mean.

2. R² and RMSE in plain language

  • R²: the fraction of variation in the target that the model explains. Range: usually 0 to 1; can be negative on out-of-sample data.
  • RMSE: the square root of the mean of the squared residuals. It has the same units as the target. A stakeholder can read "RMSE = $3.20" as "typical errors are about three dollars and twenty cents."

These are the only two metrics you need to report for Topic-3. Ridge / Lasso / adjusted R² / Q-Q plots are not in scope.

3. Why time-series data needs a chronological split

If you randomly shuffle daily stock prices and call 80% "train" and 20% "test", you have allowed the model to peek at future days while learning about earlier days. The test score will look unrealistically good — and the production model will then disappoint everyone.

Click the buttons below to see the difference visually. The blue points are training; the orange points are test. Notice how the chronological split keeps test strictly to the right of train.

4. Honest uncertainty: ± 2 standard deviations of the residuals

The introductory rule for the deployed product is: show a range, not just a point estimate. The simplest range is built from the standard deviation of the residuals on the held-out test set: 

import numpy as np
resid = y_test - y_pred
sd    = np.std(resid)
lo, hi = next_pred - 2 * sd, next_pred + 2 * sd

If the residual histogram is clearly lopsided, say so in the README: "the model under-predicts on high-price days, so the range above will tend to be wrong in that direction."

5. Overfitting in one picture

"Overfitting" means the model has memorized random wiggles in the training set that do not generalize. A flexible model can chase every training point — and look great on the training set — while producing bad predictions on data it has not seen.

The polynomial degree below controls model flexibility. Degree 1 is a straight line (linear regression). Higher degrees bend the curve more. Move the slider and watch the train and test scores diverge as the model overfits.

What to notice: The train R² always goes up (or stays flat) as degree increases. The test R² rises briefly and then collapses. That gap is the signature of overfitting. At the introductory level we use plain Linear Regression (degree 1) and accept the modest fit — we are not chasing R² at the cost of generalization.

6. Which features matter? Looking at the coefficients

A trained Linear Regression has one coefficient per feature (model.coef_). The sign of the coefficient tells you whether the feature pushes the prediction up or down, and the magnitude tells you how strongly. Below is a toy five-feature regression — drag the sliders to see how each feature contributes to the predicted output.

Watch out: Coefficient magnitudes only compare fairly when the features are on the same scale. A coefficient of 0.001 on "income in dollars" can be just as influential as a coefficient of 1000 on "indicator (0/1)". This is why we standardize features in some workflows — and why we never read raw coefficients as "feature importance" without thinking about scale.

7. End-to-end: a tiny stock-prediction walkthrough

Here is what the Topic-3 Honest Forecast product looks like in miniature — all the moving parts in one place, in the order they happen. 

# 1. Pull history (one-time, in build_sqlite.py)
import yfinance, sqlite3
df = yfinance.download("AAPL", period="3y")
con = sqlite3.connect("db/AAPL_history.sqlite")
df.to_sql("history", con, if_exists="replace")
con.close()

# 2. Read into pandas
import pandas as pd
con = sqlite3.connect("db/AAPL_history.sqlite")
df = pd.read_sql("SELECT * FROM history", con)
con.close()

# 3. Engineer two simple features (in pandas)
df["close_yesterday"] = df["close"].shift(1)
df["ma_5"]            = df["close"].rolling(5).mean()
df = df.dropna()

# 4. Chronological split (train on past, test on future)
split  = int(len(df) * 0.8)
train  = df.iloc[:split]
test   = df.iloc[split:]
X_cols = ["close_yesterday", "ma_5"]
X_train, y_train = train[X_cols], train["close"]
X_test,  y_test  = test[X_cols],  test["close"]

# 5. Fit and evaluate
from sklearn.linear_model import LinearRegression
from sklearn.metrics import r2_score, mean_squared_error
import numpy as np

model  = LinearRegression().fit(X_train, y_train)
y_pred = model.predict(X_test)
print("test R² =", r2_score(y_test, y_pred))
print("test RMSE =", np.sqrt(mean_squared_error(y_test, y_pred)))

# 6. Build the honest range for tomorrow's forecast
resid = y_test - y_pred
sd    = np.std(resid)
next_features = X_test.iloc[-1].values.reshape(1, -1)
next_pred = model.predict(next_features)[0]
lo, hi = next_pred - 2 * sd, next_pred + 2 * sd

# 7. Persist (versioned CSV)
from datetime import date
df.to_csv(f"data/derived/AAPL_features_{date.today():%Y%m%d}.csv",
          index=False)

Every Topic-3 project is a variation on this skeleton: pick a ticker, engineer one or two simple features, split chronologically, fit, evaluate on test, build the range, persist, ship.

8. When Linear Regression is the wrong tool

Plain Linear Regression assumes the relationship between features and target is roughly linear. A few situations where that fails:

  • The relationship is curved. A line cannot fit a U-shape. Solution at the introductory level: log-transform the target if it is heavily skewed, or accept the linear fit and report its limits.
  • One feature has huge outliers. Linear regression is sensitive to outliers because it minimizes squared error. Solution: clip or drop the extreme rows, or use a more robust model in a later course.
  • Heavy interaction between features. "Effect of feature A depends on feature B." Plain Linear Regression cannot capture interactions; trees and neural networks can.

The introductory habit: fit the simple model, report its weaknesses honestly, and let later courses introduce the heavier machinery.
