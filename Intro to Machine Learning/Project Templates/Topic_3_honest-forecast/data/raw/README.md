# Raw data — `prices.csv`

A **synthetic** daily price history for one ticker (`SYN`). It is generated
deterministically (fixed random seed) by
[`../../db/generate_raw.py`](../../db/generate_raw.py), so re-running that
script always produces this exact file and the tests stay reproducible. You
normally do **not** need to regenerate it — it is committed.

## Why synthetic (and how to use real data)

Downloading real market data (e.g. with `yfinance`) makes tests slow, flaky,
and network-dependent. The synthetic series is a **trend + mild seasonality +
random noise**, which is enough to make linear regression meaningful without
being trivially perfect. To use real data instead, replace `generate()` in
`db/generate_raw.py` with a `yfinance` download that returns the same three
columns.

## Table dictionary — `prices`

| Column   | Type  | Notes                                             |
|----------|-------|---------------------------------------------------|
| `ticker` | text  | Symbol; always `SYN` in this synthetic dataset.   |
| `date`   | text  | Trading day, `YYYY-MM-DD` (weekdays only).        |
| `close`  | float | Closing price = trend + seasonality + noise.      |

500 trading days. In `src/features.py` we add a numeric day index `t`
(0, 1, 2, …) as the single feature the linear model regresses `close` on.
