# Raw data — `shoppers.csv`

A **synthetic** shopper dataset. It is generated deterministically (fixed random
seed) by [`../../db/generate_raw.py`](../../db/generate_raw.py), so re-running
that script always produces this exact file and the tests stay reproducible. You
normally do **not** need to regenerate it — it is committed.

## Why synthetic (and how to use a real dataset)

A real customer dataset would work too, but a synthetic one lets us plant a
clear structure so the unsupervised methods have something to rediscover. Every
shopper secretly belongs to one of three **segments**, each with its own typical
numeric profile *and* its own basket habits. To use real data, replace
`generate()` in `db/generate_raw.py` with a loader that returns the same
columns (the five numeric features plus the six `bought_*` flags).

## Provenance

- **Source:** Synthetic shopper-segment data generated by `db/generate_raw.py`.
- **License:** Public domain (synthetic — no real customers).
- A copy of this provenance is written into a `provenance` table when the
  SQLite database is built.

## Table dictionary — `shoppers`

| Column               | Type | Notes                                              |
|----------------------|------|----------------------------------------------------|
| `age`                | int  | Customer age (18–90).                              |
| `annual_income`      | int  | Yearly income in dollars.                          |
| `spending_score`     | int  | Store-assigned spending score (1–100).             |
| `visits_per_month`   | float| Average store visits per month.                    |
| `avg_basket_value`   | float| Average dollars spent per visit.                   |
| `bought_groceries`   | int  | 1 = bought groceries, else 0.                      |
| `bought_electronics` | int  | 1 = bought electronics, else 0.                    |
| `bought_clothing`    | int  | 1 = bought clothing, else 0.                       |
| `bought_home`        | int  | 1 = bought home goods, else 0.                     |
| `bought_toys`        | int  | 1 = bought toys, else 0.                           |
| `bought_beauty`      | int  | 1 = bought beauty products, else 0.                |
| `segment`            | int  | **Ground truth.** Hidden group 0/1/2 — not used by |
|                      |      | the algorithms, only to check and colour results.  |

900 rows (three segments of 300). The five numeric features form three clusters;
the `bought_*` flags carry segment-specific co-purchase patterns for the
association-rule miner.
