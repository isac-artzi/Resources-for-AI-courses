# Raw data — `emails.csv`

A **synthetic** labelled spam / ham (not-spam) email corpus. It is generated
deterministically (fixed random seed) by
[`../../db/generate_raw.py`](../../db/generate_raw.py), so re-running that
script always produces this exact file and the tests stay reproducible. You
normally do **not** need to regenerate it — it is committed.

## Why synthetic (and how to use a real corpus)

Real spam datasets are large and awkward to redistribute. The synthetic corpus
draws each message from three word lists — spammy, hammy, and neutral — with a
small amount of cross-class "leakage" so the classes are realistic rather than
trivially separable. To use a real corpus instead, replace `generate()` in
`db/generate_raw.py` with a loader that returns the same two columns.
