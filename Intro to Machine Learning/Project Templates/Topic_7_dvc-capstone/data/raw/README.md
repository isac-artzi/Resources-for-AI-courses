# Raw data — `houses.csv`

A **synthetic** house-price dataset. It is generated deterministically (fixed
random seed) by [`../../db/generate_raw.py`](../../db/generate_raw.py), so
re-running that script always produces this exact file and the tests stay
reproducible. You normally do **not** need to regenerate it — it is committed.

## Why synthetic (and how to use a real dataset)

A real housing dataset would work too, but a synthetic one lets us plant the
exact problems feature engineering is meant to solve: a categorical column whose
price premiums are **not** in alphabetical order (so naive label-encoding fails)
and two **right-skewed** size columns (so a log transform helps). To use real
data, replace `generate()` in `db/generate_raw.py` with a loader that returns
the same columns plus a numeric `price` target.

## Table dictionary — `houses`

| Column         | Type | Notes                                                 |
|----------------|------|-------------------------------------------------------|
| `neighborhood` | text | One of five areas; premiums are NOT alphabetical.     |
| `sqft`         | int  | Living area in square feet (right-skewed).            |
| `lot_size`     | int  | Lot size in square feet (heavily right-skewed).       |
| `year_built`   | int  | Construction year (1950–2023).                        |
| `bedrooms`     | int  | Number of bedrooms (1–6).                             |
| `bathrooms`    | int  | Number of bathrooms (1–4).                            |
| `garage`       | int  | 1 = has a garage, 0 = none.                           |
| `condition`    | int  | Condition rating (1–5).                               |
| `price`        | int  | **Target.** Sale price in dollars.                    |

1500 rows. Price follows a hidden rule: a neighborhood premium plus linear
effects of size, rooms, garage, and condition, minus age, plus noise — so no
model is perfect, and good features matter.
