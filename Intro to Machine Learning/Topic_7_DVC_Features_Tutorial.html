Topic 7 · DVC, Feature Engineering & Deployment

Topic 7 ties the term together. You learn DVC (Git for data & models), apply an introductory feature-engineering toolkit in pandas, and ship a v1.0.0 release of one of your earlier products. This page visualizes both the DVC pipeline DAG and the feature-engineering toolkit.

1. DVC = Git for data & models

Git is great for source code but bad for large binary files: every new version of a 200 MB CSV is stored in full in the repo history, bloating clones and making diffs useless. DVC fixes this by keeping a tiny "pointer" file (foo.csv.dvc) in Git and storing the actual bytes on a separate remote (S3, GCS, SSH, etc.). 

# one-time setup
git init
dvc init
dvc remote add -d storage s3://my-bucket/path

# track a big file
dvc add data/raw/orders.csv
git add data/raw/orders.csv.dvc .gitignore
git commit -m "add raw"
dvc push    # uploads the bytes to the remote

2. A DVC pipeline as a small DAG

A dvc.yaml file declares "stages" that depend on each other. dvc repro walks this DAG and re-runs only the stages whose declared inputs (deps) have changed. Hover over each box below to see the inputs and outputs.

The arrow direction is data flow. If you edit src/featurize.py, only the featurize and train stages will re-run — prepare is reused from the cache because its inputs haven't changed.

4. Compare baseline vs engineered (be honest)

Good practice rewards honest reporting of whether your engineered features actually helped. Train the baseline and the engineered model on the same training set, evaluate on the same held-out test set, and put the numbers in a table:

ModelTest R² / AUCTest RMSE / F1Comment
Baseline (no new features)0.713.42—
Engineered0.743.18Small improvement; explain why.

If the engineered model is worse, say so in the README — don't quietly drop the comparison. Negative results are part of professional product work.

7. dvc.yaml side-by-side with the Python it runs

Every stage in dvc.yaml is just a shell command. The table below pairs each stage with the Python module it invokes, showing how the configuration glues to the code.

Stagedvc.yamlWhat the Python does
preparepython src/prepare.py data/raw/orders.csv data/interim/clean.csvReads raw CSV, drops bad rows, writes interim CSV.
featurizepython src/featurize.py data/interim/clean.csv data/derived/feat.csvApplies log / binning / one-hot, writes derived CSV.
trainpython src/train.py data/derived/feat.csv models/m.pklSplits, fits, saves model and metrics.

Why this matters: you can re-run the whole pipeline from scratch with one command (dvc repro). That is reproducibility — the discipline that turns "my notebook" into "our product."

9. Two-tier deployment in pictures

The optional tier splits the application into a Streamlit UI and a FastAPI service. The diagram below shows the request path. Hover any box to see what runs there and what it returns.

What you gain

  • Independent deploys — ship a new model without touching the UI.
  • Shared scoring — the same /predict endpoint can power a CLI, a Slack bot, another web app.
  • Cleaner ownership — UI changes vs model changes flow through different PRs.

What you give up

  • Network latency on every request.
  • One new failure mode: the service can be down or unreachable.
  • More deployment plumbing (two free-tier hosts instead of one).

10. The capstone checklist

Before you ship, walk down this list. Each item matters; missing any one is a visible gap in your release.

  • DVC remote configured and at least one dvc push succeeded.
  • dvc.yaml with 3+ stages and dvc repro works on a fresh clone.
  • 3+ engineered features from the introductory toolkit, applied in pandas before training.
  • Baseline-vs-engineered comparison table in the README, with honest commentary if engineering did not help.
  • Trained Pipeline persisted with joblib and loaded by the Streamlit app with @st.cache_resource.
  • Live Streamlit Cloud URL pinned to the top of the README.
  • (Optional) FastAPI service deployed on Render with the Streamlit UI calling it over HTTP and the trade-off documented.
