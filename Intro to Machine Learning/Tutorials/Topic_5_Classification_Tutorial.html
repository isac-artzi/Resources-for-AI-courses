2. Why scaling matters for kNN and SVM (but not trees)

k-Nearest Neighbors decides each point's class by looking at its closest neighbors. "Closest" is a distance, so if one feature is measured in dollars (range 0 — 100,000) and another in years (range 0 — 60), the dollar feature will dominate every distance calculation. A StandardScaler brings both features to the same numeric scale so each gets a fair say.

Decision trees, in contrast, split on one feature at a time — "is age < 35?" — so the numeric scale of the OTHER features is irrelevant. Trees are scale-invariant.

3. Reading a confusion matrix

For a binary classifier, the matrix has four cells:

predicted 0 (negative)predicted 1 (positive)
actual 0TN — correct rejectionFP — false alarm
actual 1FN — missed positiveTP — caught positive

From these four numbers you can compute every standard metric:

  • Accuracy = (TP + TN) / N
  • Precision = TP / (TP + FP)
  • Recall = TP / (TP + FN)
  • F1 = harmonic mean of precision and recall

4. The ROC curve — pick a threshold, see the trade-off

Most classifiers don't just output a class — they output a probability. We turn that into a class by comparing to a threshold. Lower the threshold (be more eager to flag positives) and recall goes up but precision goes down. The ROC curve plots true-positive rate vs false-positive rate across all thresholds.

5. The effect of k on a kNN boundary

k-Nearest Neighbors votes among the k closest training points. A small k makes the boundary jagged (high variance); a large k makes it smooth (high bias). Drag the slider to see for yourself.

What to notice: at k=1 the boundary memorizes every training point — including noise. At k=21 it almost ignores the data and picks a smooth average. Real life is somewhere in between; cross-validation lets the data tell you what k to use, but the default of 5 is a reasonable starting point.

6. Decision tree: depth controls overfitting

A decision tree splits the feature space along one axis at a time. max_depth caps how deeply it can split — deeper trees fit more (and overfit more). Drag the slider to see how a tree's boundary changes with depth on the same dataset.

7. Feature scaling — kNN with and without

When features have wildly different units (dollars vs years), kNN's distance computation is dominated by the large-magnitude feature. StandardScaler brings both features onto comparable scales so each contributes fairly. 9. Side-by-side classifier comparison

At the introductory level you compare classifiers by putting their test-set metrics in a single table. Below is a worked example — notice that different products would ship different classifiers from these same numbers, depending on which metric matters most.

ClassifierAccuracyPrecisionRecallF1AUC
kNN (k=5)0.8820.810.740.770.91
SVM (default RBF)0.8950.830.780.800.93
Decision Tree (depth 5)0.8710.790.810.800.88

Reading this table: if you care most about overall AUC, ship the SVM. If you care most about catching positives at the cost of some false positives, the tree's slightly higher recall might win. At the introductory level you should make this decision and defend it in writing — not just present the numbers.
